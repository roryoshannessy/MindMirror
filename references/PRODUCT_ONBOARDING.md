# MindMirror Product Onboarding

Last updated: June 2, 2026

Purpose: keep onboarding focused on the first real product moment, not generic account setup.

## Principle

MindMirror should not ask users to build a journaling habit before they feel value.

The first session should do one thing well:

```text
Start with one thought.
Save it.
Show the possible loop.
Ask the next question.
```

## After Signup Flow

### 1. What keeps coming back?

Ask for one thought, worry, decision, loop, or feeling.

Keep the screen sparse. Voice first, text always available.

### 2. When does it hit?

Optional later step. Do not block the first reflection.

Useful options:

- Morning
- During work
- After work
- Before bed
- When I am alone

### 3. What should MindMirror catch?

Optional later step after the first mirror.

Useful options:

- Overthinking
- Avoidance
- Pressure
- Self-blame
- Work loops
- Relationship loops

### 4. Your mirror is ready

After the first saved reflection, show:

- possible loop
- cost
- evidence
- next question
- smallest action

## Evidence Ladder

Use truthful unlocks:

- 1 reflection: first mirror
- 3 reflections: early signals
- 10 reflections: pattern map

Language:

```text
MindMirror gets sharper as it has more evidence.
```

## Notification Timing

Do not ask for reminders during signup.

Ask only after the user has seen a useful mirror:

```text
Want MindMirror to check in when this usually comes back?
```

Options:

- Morning
- After work
- Before bed
- Not now

## What To Avoid

- Generic wellness onboarding.
- Mood tracking before the first mirror.
- Streak pressure.
- Notification permission before value.
- A long questionnaire before the user saves a thought.
